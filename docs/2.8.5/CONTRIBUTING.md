# Contributing to ZION Blockchain

Thank you for your interest in contributing to ZION! We welcome contributions from developers, testers, translators, and community members worldwide.

## üéØ Ways to Contribute

### 1. Report Bugs

Found a bug? Please help us improve by reporting it:

1. Check [existing issues](https://github.com/Zion-TerraNova/Zion-TestNet2.8.5/issues) to avoid duplicates
2. Create new issue with template
3. Include:
   - Platform (Linux/macOS/Windows) and version
   - Binary version (`./zion-node --version`)
   - Steps to reproduce
   - Expected vs actual behavior
   - Logs (if applicable)

### 2. Suggest Features

Have an idea? We'd love to hear it:

1. Open [Discussion](https://github.com/Zion-TerraNova/Zion-TestNet2.8.5/discussions)
2. Describe use case and benefits
3. Consider technical feasibility
4. Engage with community feedback

### 3. Improve Documentation

Documentation is crucial:

- Fix typos or unclear explanations
- Add examples or tutorials
- Translate to other languages
- Create video guides

**Process:**
1. Fork repository
2. Edit Markdown files in `docs/`
3. Submit Pull Request

### 4. Test & Provide Feedback

Help us test on different platforms:

- Test binaries on your OS/hardware
- Try different mining configurations
- Test Sacred Library API
- Report performance metrics
- Suggest UX improvements

### 5. Develop SDK Examples

Create useful examples for community:

- Python scripts for automation
- JavaScript dApps
- Integration tutorials
- Sacred Library apps (meditation, education)

## üîß Development Guidelines

### Code Style

**Python (if contributing to examples):**
- Follow PEP 8
- Use type hints
- Add docstrings
- Max line length: 100

**JavaScript/TypeScript:**
- Follow ESLint rules
- Use TypeScript for new code
- Add JSDoc comments

### Commit Messages

Format: `TYPE: Description`

**Types:**
- `FIX`: Bug fix
- `FEAT`: New feature
- `DOCS`: Documentation
- `TEST`: Testing
- `REFACTOR`: Code refactoring
- `STYLE`: Formatting
- `CHORE`: Maintenance

**Examples:**
```
FIX: Correct balance calculation in wallet.py
DOCS: Add mining guide for Windows users
FEAT: Add Sacred Library search filter
```

### Pull Request Process

1. **Fork** repository
2. **Create branch** from `main`:
   ```bash
   git checkout -b feature/your-feature-name
   ```
3. **Make changes** with clear commits
4. **Test locally** (if applicable)
5. **Submit PR** with description:
   - What changed
   - Why it's needed
   - How to test
   - Related issues (if any)
6. **Respond to feedback** from maintainers
7. **Wait for approval** and merge
