Mining Algorithms

Supported Algorithms (ASIC-Resistant)

1. Cosmic Harmony (Native)

Type:              Multi-hash PoW
Memory:            Moderate (512 MB minimum)
CPU Performance:   19,000 H/s (Python) ‚Üí 100,000-500,000 H/s (native)
GPU Performance:   850 MH/s (RTX 3080 OpenCL)
ASIC Resistance:   95% higher than SHA256
Power Efficiency:  2.66 MH/W

Default algorithm. Native ZION implementation combining multiple hash functions (SHA-256, Keccak-256, Blake2b, SHA3-512) with state mixing for ASIC resistance.

2. RandomX (CPU-Optimized)

Type:              CPU-optimized PoW
Memory:            2 GB+ recommended
CPU Performance:   80,000 H/s (baseline) ‚Üí 2,000-10,000 H/s (native)
ASIC Resistance:   Extreme (designed for general-purpose CPUs)
Power Efficiency:  Optimal on modern x86-64 CPUs

Based on Monero's RandomX, optimized for CPU mining with large memory requirements that make ASIC development economically unfeasible.

3. YesCrypt (Memory-Hard)

Type:              Memory-hard PoW
Memory:            1 GB+ required
CPU Performance:   7,000 H/s (baseline) ‚Üí 500-2,000 H/s (optimized)
Energy Efficiency: Very high (low power consumption)
ASIC Resistance:   High (memory bandwidth bottleneck)

Memory-hard algorithm that provides excellent ASIC resistance through high memory bandwidth requirements. Eco-friendly bonus in consciousness mining.

4. Autolykos v2 (GPU-Friendly)

Type:              Memory-hard GPU PoW
Memory:            4 GB+ GPU VRAM
GPU Performance:   25-30 MH/s (AMD RX 5600 XT)
                   170,000 H/s (baseline) ‚Üí 10,000-50,000 H/s (optimized)
ASIC Resistance:   High (memory access patterns)
Power Efficiency:  Good on modern GPUs

Ergo's GPU algorithm providing memory-hard PoW that's efficient on modern GPUs while maintaining ASIC resistance.

Algorithm Selection Priority

  1. Cosmic Harmony: Default, best for native ZION miners
  2. RandomX: CPU miners, high ASIC resistance
  3. YesCrypt: Energy-efficient mining, eco-friendly
  4. Autolykos v2: GPU miners with 4GB+ VRAM

Security Architecture

Critical Security Measure: Public repository distributes only compiled binaries to protect 15.78B ZION premine addresses and genesis logic.

Security Benefits

  • ‚úÖ Premine addresses not visible in source code
  • ‚úÖ Genesis block logic protected from reverse engineering
  • ‚úÖ Algorithm implementations obfuscated
  • ‚úÖ Private key derivation paths secured
  • ‚úÖ Minimal attack surface (no Python runtime in production)
