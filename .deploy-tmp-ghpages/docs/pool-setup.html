Pool Setup

Run a mining pool for the ZION TestNet: configuration, Stratum, and monitoring.

Overview

A pool aggregates miners to reduce variance. You will run a Stratum server, track shares, and pay miners proportionally.

Requirements

  • Server with stable network and static IP
  • SSD storage, 8–16 GB RAM, multi-core CPU
  • Public ports open (e.g., 3333/tcp for Stratum)

Configuration

  1. Run a full node in testnet mode and keep it in sync.
  2. Configure Stratum endpoint (hostname, port, difficulty targeting).
  3. Implement share validation and block submission to node.
  4. Set payout scheme (PPLNS/PPS) and payout intervals.

Stratum Protocol

Protocol details are evolving on testnet. Start with standard Stratum semantics and adapt to testnet updates. Announcements will be in the repository.

Monitoring

  • Track hashrate, shares, orphan rate, and connected miners.
  • Alert on stale connections and high reject rates.
  • Publish a simple pool status page for miners.
